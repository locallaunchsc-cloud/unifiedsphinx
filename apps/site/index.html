Core modules

One runtime. Every layer of defense.

Traditional security wasn't built for agents that write code, click buttons, and call APIs at machine speed. UnifiedSphinx was.

Policy Engine

Rule-based + scored decisions for every event. Allow, alert, or block in under 50ms. Shipping with detectors for path probing, SQLi, rate abuse, and bot UAs.

Risk Scoring

0–100 composite score per request blending behavior, reputation, and content signals. Thresholded actions so noisy alerts become decisions.

Prompt-Injection Guard

Detects attempts to hijack your LLM calls — jailbreaks, instruction override, and tool-abuse payloads. Plugs directly into your agent pipeline.

PII & Secret Leak Detection

Regex + entropy scanners catch emails, tokens, SSNs, and API keys in real-time payloads before they ship to third-party models.

Drop-in Sentinel Snippet

One <script> tag on any site streams events — page views, form submits, risky URL params — into your Sphinx runtime with zero config.

Real-time Dashboard

Dark-mode command center: live event stream, security score, alert counters, and drill-downs per site. Built for operators, not compliance theater.

How it works

Three wires. Machine-speed defense.

  1. 01

    Drop the snippet

    Add sentinel.js to your site or wrap your agent. Events start flowing to your Sphinx runtime in seconds.

    <script src="sphinx/sentinel.js"
  data-site-id="my-site"></script>
  2. 02

    Runtime scores every event

    The policy engine + risk model grade each event and return a decision: allow, alert, or block — before the action completes.

    POST /events → { decision: "block",
  score: 87, reason: "prompt_injection" }
  3. 03

    Operators see everything

    Live dashboard, plain-English alerts, and an append-only audit log. Your team responds in minutes, not days.

    [ALERT] Agent attempted exfiltration
→ blocked @ 03:14:22 UTC
